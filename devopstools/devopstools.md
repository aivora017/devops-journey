. CI/CD pipeline

CI - Continious integration
CD - Continious Development / Delivery

developer -> github -> build -> test -> production

. Container (Docker) - provides isolation as well as provides build images packed with dependencies and config that is easy to run on other servers

Kubernetes :- It is a container orceshtrator that manages to upscle or downscale resorces.
terraforam :- it is an infrastructure provisiong system that is used to maintain the config or state of servers in a cluster.
Ansible :- it is used to automate and post provisiong the server after server is provisioned by terraform programs require different dependencies and congifs that is maintained by ansible.
prometheus :- it stores metric data like memory usage, process usage and other such data
grafana :- it is used to visualize the data stored in prometheus
