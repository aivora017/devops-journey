##git

it is used to manage codebases without creating overlaps and conflicts.

the steps for git are

1. initialize a git repository
2. configure files to track
3. git track changes
4. stage changes
5. commit changes

commands

- yum intall git -> it is used to intall
- git -v -> it is use to know the version of git
- git init -> it used to inintialize a git repo
- git status -> it used to check the status of the repo give all tracked and untracked files and ready to cmmit files
- git add -> it is used to stage files for the commit
- git commit -m "comments" -> it commits the file with a comment

- git remote add <name> <url>  -> it creates a remote connection to the github url repo name can be different in your system no required to be same as repo.
- git push -u github master -> when code it pushed to git hub repo it is pushed to the master node by default
- git clone url -> it is used to clone the git repo from the github to our own machine
- git pull -> it is used to update the changes from github repo to repo in local machine
- git remote -v -> it tells about all the remote connections available
